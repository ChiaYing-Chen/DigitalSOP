# BPMN.io 符號圖解指南 (Cheat Sheet)

這份指南詳細說明了本系統支援的所有 BPMN 符號及其用途。

## 1. 任務與活動 (Activities)

這是流程圖的核心，代表「誰做了什麼動作」。本系統支援點擊以下任務來記錄執行狀況。

### 🟦 基本任務
- **一般任務 (Task)**：最通用的任務類型。
- **使用者任務 (User Task)** 👤：需要人員在系統上操作的任務（例如：登入審核）。
- **手動任務 (Manual Task)** ✋：純人工執行的物理動作，不透過系統（例如：搬運貨物）。

### ⚙️ 系統自動化任務
- **服務任務 (Service Task)** ⚙️：系統自動執行的背景作業。
- **腳本任務 (Script Task)** 📜：執行一段程式碼腳本。
- **業務規則任務 (Business Rule Task)** 📋：依據規則進行決策或計算。

### ✉️ 訊息相關任務
- **發送任務 (Send Task)** ✉️：發送訊息給外部（例如：寄信）。
- **接收任務 (Receive Task)** 📩：等待接收外部訊息（例如：等回信）。

### 🔄 其他活動
- **呼叫活動 (Call Activity)**：呼叫並執行另一個獨立的流程（子流程）。
- **展開的子流程 (Expanded Subprocess)**：將一組相關的任務包在一個大框框內，視為一個整體階段。

## 2. 資料物件 (Data)

本系統支援對資料物件設定 **超連結**，點擊後可開啟相關文件或網頁。

### 📄 資料物件 (Data Object Reference)
- **圖示**：一張折角的紙。
- **用途**：代表流程中產生或使用的文件、報告、Email。
- **功能**：若有設定超連結，點擊可開啟外部網頁。

### 🗄️ 資料儲存 (Data Store Reference)
- **圖示**：資料庫圓柱體。
- **用途**：代表資料被持久化儲存的地方（例如：資料庫、檔案櫃）。
- **功能**：若有設定超連結，點擊可開啟外部系統連結。

## 3. 閘道器 (Gateways)

流程的分岔路口，用來決定路徑。

### ❌ 互斥閘道 (Exclusive Gateway / XOR)
- **圖示**：空心菱形或有 X。
- **意義**：**單選題**。只能走其中一條路（例如：審核 通過/不通過）。

### ➕ 平行閘道 (Parallel Gateway / AND)
- **圖示**：內部有 + 號。
- **意義**：**同時進行**。所有分岔的路徑都會同時開始執行。

### ⭕ 包容閘道 (Inclusive Gateway / OR)
- **圖示**：內部有圓圈 O。
- **意義**：**複選題**。可以走一條或多條路，只要條件符合的路徑都會執行。

### ✴️ 複雜閘道 (Complex Gateway)
- **圖示**：內部有星號 *。
- **意義**：處理複雜的合併或分支邏輯。

### 💠 事件驅動閘道 (Event-based Gateway)
- **圖示**：內部有雙圈圖示。
- **意義**：路徑的選擇取決於「哪個事件先發生」（例如：先收到錢 vs 先超過期限）。

---

## 4. 事件 (Events)

### 🟢 開始事件 (Start Event)
- **圖示**：細線圓圈。
- **意義**：流程的起點。

### 🟡 中間事件 (Intermediate Event)
- **圖示**：雙線圓圈。
- **意義**：流程中發生的事件（例如：等待時間、收到訊號）。

### 🔴 結束事件 (End Event)
- **圖示**：粗線圓圈。
- **意義**：流程的終點。

---

## 5. 泳道與連接線

- **池 (Pool)**：代表完整的組織（如：公司）。
- **道 (Lane)**：代表組織內的部門或角色（如：經理、員工）。
- **順序流 (Sequence Flow)** ➡️：實線箭頭，代表執行順序。
- **訊息流 (Message Flow)** ⇢：虛線箭頭，代表不同組織間的溝通。
